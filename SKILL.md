# IMU-Distilled Gait Experiment Result Analysis Skill

## 1. 目标
这个 Skill 专门用于分析“将 RobustCap 的 IMU 信息通过蒸馏方式融入 Gait 模型”的实验结果，而不是泛泛分析一般步态识别结果。

它的目标是：
1. 分析蒸馏后的 gait 学生模型，是否相比原始 gait baseline 有稳定提升。
2. 判断提升到底更像来自：
   - IMU 带来的运动学/姿态先验，
   - 额外监督信号带来的训练正则化，
   - 还是仅仅来自训练技巧变化。
3. 判断蒸馏是否真的让 gait 特征学到了更强的结构信息、时序一致性或跨条件鲁棒性。
4. 识别“有提升但解释不成立”的情况，例如：
   - Rank-1 涨了，但跨视角没有改善
   - 某些条件有效，某些条件无效
   - 只有总体涨，关键困难子集没涨
   - 提升主要来自更长训练或更强增强，而不是蒸馏本身
5. 输出适合：
   - 自己排查实验
   - 给导师组会汇报
   - 写进论文结果分析
   的结构化文字。
6. 给出下一步最值得补做的验证实验。

---

## 2. 背景约束
分析时默认基于以下研究背景：

1. RobustCap 是一个融合单目图像与稀疏 IMU 的实时人体动作捕捉系统，仓库 README 明确说明训练入口是 `net/sig_mp.py`，并在 live demo 中使用 6 个 Xsens Dot IMU 与单目摄像头。  
2. RobustCap 论文强调其核心是双坐标策略：一条分支在人体根坐标系中利用 IMU，更适合姿态估计；另一条分支将 IMU 转到相机坐标系后与视觉信息结合，更适合与视觉结果融合。  
3. `sig_mp.py` 的在线前向中包含多个具有蒸馏价值的中间表征，例如：
   - `j3dr_i`：惯性分支产生的 3D 关节相关表征
   - `vr`：与速度/平移估计相关的惯性输出
   - `j3dc`：视觉相关的 3D 关节表征
   - `poseg6d`：姿态 6D 表征
   - `contact`：足部接触相关表征
   - `pc`：平移或位置校正相关表征  
4. 你的目标不是在 gait 推理阶段直接输入 IMU，而是在训练阶段将 RobustCap 教师侧的 IMU 相关信息蒸馏到 gait 学生模型中，让学生只依赖视觉模态也能学到更强的运动结构知识。
5. 因此，实验分析的重点不是“多模态融合是否有效”，而是：
   - 蒸馏是否把教师中的 IMU 先验迁移给了 gait 学生；
   - 这种迁移是否真的提升了步态识别，而不只是让训练更复杂。

以上背景若与用户当前实现有出入，以用户的实际代码和实验设定为准。:contentReference[oaicite:1]{index=1}

## 3. 适用场景

### 3.1 模型对比
- 原始 SkeletonGait / SkeletonGait++ vs 加入 IMU 蒸馏后的版本
- 纯视觉 gait baseline vs IMU-knowledge distilled gait
- 不同蒸馏目标之间的比较
- 不同蒸馏权重、蒸馏层、蒸馏阶段之间的比较

### 3.2 消融实验
- 不蒸馏 vs 蒸馏
- 只蒸馏单一教师表征 vs 蒸馏多个教师表征
- 只蒸馏中间特征 vs 蒸馏输出空间
- 只蒸馏姿态相关信号 vs 再加入接触/平移相关信号
- 冻结 backbone vs 联合训练
- 不同 loss 权重与组合方式的比较

### 3.3 数据与协议对比
- SUSTech1K、GREW、Gait3D、OU-MVLP 等数据集上的表现
- 跨视角、跨场景、跨外观条件下的表现
- 室内与室外、简单与复杂背景、遮挡与非遮挡条件下的表现

### 3.4 训练过程分析
- 蒸馏是否更难收敛
- 是否出现 teacher 约束过强导致主任务受损
- 是否出现早期有效、后期退化
- 是否出现表示冲突、loss 不平衡、训练不稳定

## 4. 不适用场景
以下情况不要强行下强结论：

1. 用户没有提供可比较的 gait 识别结果。
2. 用户只给了 teacher loss 下降，却没有识别指标结果。
3. 用户没有说明 baseline、数据集、协议或训练差异。
4. 用户想让模型夸大蒸馏效果或掩盖退化。
5. 用户当前只是要解释代码报错，而不是分析实验结果。

如果输入不足，应先说明限制，再做最保守的分析。

## 5. 分析时优先关注的输入

### 5.1 核心输入
优先识别：
- 学生模型是什么（如 SkeletonGait++）
- baseline 是什么
- 蒸馏后模型改了什么
- 数据集与协议
- 核心识别指标
- 训练配置差异
- 是否公平对照

### 5.2 与 IMU 蒸馏尤其相关的输入
如果用户提供了，优先纳入分析：
- 教师表征来自 RobustCap 的哪一部分
- 蒸馏目标是什么：
  - feature distillation
  - logit distillation
  - representation alignment
  - temporal consistency distillation
  - pose/contact/translation-related distillation
- 蒸馏层位置
- 蒸馏损失形式
- 蒸馏权重
- 训练阶段是否分阶段进行
- 是否只在训练中使用 teacher
- 推理时是否完全不使用 IMU
- 不同蒸馏目标对应的结果差异
- 失败案例或特定困难条件下的表现

### 5.3 输入不足时的规则
若缺少对照组、协议或训练设定：
1. 不要把结果写成“IMU 知识已成功迁移”。
2. 只能陈述观察到的结果变化。
3. 所有机制解释都必须使用保守措辞。
4. 默认建议补做公平对照与蒸馏消融实验。

## 6. 本方向专用分析逻辑
分析“IMU 蒸馏到 gait”时，默认围绕以下六个问题展开：

### 6.1 蒸馏是否真的有效
核心判断：
- 蒸馏后学生模型是否稳定优于视觉 baseline
- 提升是否跨多个 protocol / subset 都存在
- 还是只在单一设置下偶然提升

### 6.2 提升更像来自“知识迁移”还是“训练正则化”
必须区分：
- 是因为学生真的学到了 IMU 相关的运动结构先验
- 还是因为多了一个辅助损失，训练更稳定
- 还是因为训练时间更长、参数更多、调参更充分

### 6.3 蒸馏学到的更像哪类知识
结合 teacher target 判断：
- 若蒸馏 `j3dr_i` / `j3dc` / `poseg6d` 更有效，可能更接近姿态结构知识迁移
- 若蒸馏 `contact` / `vr` / `pc` 更有效，可能更接近时序运动学、足部接触或位移一致性约束
- 若多种 target 联合效果更好，才更能支持“IMU 复合先验”迁移成功

### 6.4 提升体现在哪类 gait 难点上
优先判断提升是否体现在：
- 跨视角
- 复杂外观变化
- 背景干扰
- 时序不稳定
- 姿态变化较大
- silhouette / skeleton map 质量不稳定的样本

若这些难点没有明显改善，则需要谨慎解释“IMU 先验”的作用。

### 6.5 蒸馏是否与 gait 主任务冲突
主动检查：
- 主识别指标涨了还是掉了
- teacher loss 很好看，但 Rank-1 / mAP 没涨
- 蒸馏权重一大就退化
- 说明 teacher 约束可能过强，或 teacher/student 空间不匹配

### 6.6 是否真的符合研究叙事
本方向最核心的研究叙事应是：
“IMU 在 RobustCap 中提供了人体运动的额外先验；这些先验可以作为教师知识迁移给纯视觉 gait 学生，使其在不依赖 IMU 推理的前提下学到更强的结构与时序表征。”

分析时必须判断当前结果是否真的支持这句话，而不是只支持其中一小部分。

## 7. 标准分析流程

### Step 1: 明确实验问题
先把实验归类：
- 验证 IMU 蒸馏是否提升 gait 识别
- 验证哪种 teacher target 更有效
- 验证哪种蒸馏方式更有效
- 验证蒸馏提升的是结构性鲁棒性还是普通精度
- 验证蒸馏是否值得其训练复杂度

### Step 2: 明确对比对象
必须说清：
- baseline 是哪一个 gait 模型
- 新模型相比 baseline 改了什么
- teacher 是否仅训练时出现
- 推理时是否完全保持纯视觉
- 除蒸馏外，其余训练设置是否一致

### Step 3: 提取关键结果
优先整理：
- Rank-1
- mAP
- 各 view / condition / subset 的结果
- 训练稳定性
- 蒸馏 loss 与主任务 loss 的关系
- 参数量、训练成本、推理成本
- 若有可视化，关注困难样本变化

### Step 4: 先做事实层对比
只做客观描述：
- 整体是否提升
- 提升幅度多大
- 哪些条件下提升明显
- 哪些条件下无明显收益
- 是否有退化项
- 是否增加训练代价

### Step 5: 再做机制层解释
优先从以下角度解释：
- 是否更像结构知识迁移成功
- 是否更像时序一致性增强
- 是否更像 teacher regularization
- 是否存在 teacher/student 表征不匹配
- 是否可能只是 baseline 调参不充分造成的假提升

### Step 6: 检查异常与风险
主动检查：
- 识别指标提升很小
- 单次实验无法说明问题
- 只有个别视角/个别子集提升
- 主任务下降但蒸馏项变好
- 跨数据集泛化不一致
- 提升不足以支撑更复杂训练

### Step 7: 形成分级结论
每次都区分：
1. 可以确认的事实
2. 合理推断
3. 当前还不能证明的内容

### Step 8: 给出下一步实验建议
建议必须直接服务于：
- 验证蒸馏是否有效
- 验证蒸馏学到了什么
- 验证提升是不是稳定的
- 验证哪类 teacher 信号最值钱

### 8.3 关键结果总结
先用几句话总结：
- 是否整体提升
- 提升主要出现在哪些数据集/协议/条件
- 是否存在退化

### 8.4 蒸馏贡献分析
必须单独分析：
- teacher target 可能提供了什么信息
- 学生模型可能学到了什么
- 提升更像来自结构先验、时序约束，还是训练正则化

### 8.5 主要提升点
例如：
- Rank-1 提升
- mAP 提升
- 跨视角更稳
- 困难条件更稳

### 8.6 主要退化点
例如：
- 某些协议下降
- 蒸馏收益很弱
- 提升不稳定
- 训练成本明显上升
- 主任务与蒸馏项冲突

### 8.7 原因分析
从以下角度优先解释：
- 结构知识迁移
- 时序一致性约束
- teacher/student 表征不匹配
- 蒸馏目标选择不合适
- 蒸馏权重过大或过小
- teacher 信号质量不足
- 训练过程正则化效应

### 8.8 风险与不确定性
明确指出：
- 是否可能只是单次波动
- 是否缺少公平对照
- 是否还缺少跨数据集验证
- 是否还不能证明学生真正学到了 IMU 语义

### 8.9 是否支持当前假设
明确回答：
- 支持
- 部分支持
- 暂不支持

## 9. 本方向专用检查项
每次分析至少检查以下问题：

1. 蒸馏后学生模型是否稳定优于 baseline
2. 提升是否跨多个 protocol / subset 都存在
3. 提升是否更集中在困难样本、跨视角或结构变化明显的场景
4. 不同 teacher target 的收益是否不同
5. 姿态类 target 是否比接触/平移类 target 更有效
6. 多目标联合蒸馏是否优于单目标蒸馏
7. 主任务指标与蒸馏 loss 是否一致改善
8. 推理成本是否保持不变或基本不变
9. 增益是否足以覆盖训练复杂度上升
10. 是否真的能支撑“IMU 知识迁移到纯视觉 gait”这一叙事

---

## 10. 异常情况处理

### 10.1 蒸馏后整体不升反降
默认判断方向：
- teacher target 与 gait 任务不匹配
- 蒸馏约束过强
- teacher 表征与学生特征空间差异过大
- 蒸馏目标干扰了主识别目标

### 10.2 只有某一小类实验有效
默认表述：
- 说明蒸馏可能只对特定条件有效
- 还不能说是通用提升
- 需要补更多协议与数据集验证

### 10.3 teacher loss 很漂亮，但识别指标没涨
默认表述：
- 说明学生更好地拟合了 teacher 表征，但未必转化为 gait 判别收益
- 不宜直接宣称蒸馏成功

### 10.4 Rank-1 涨了，但 mAP 没明显变化
默认表述：
- 可能 top-1 判别略有增强，但整体检索质量改善有限
- 不适合直接写成“整体表征显著增强”

### 10.5 跨视角没改善，但总体有轻微提升
默认表述：
- 当前结果更像普通监督收益，而不是明显的运动结构迁移
- 需要谨慎使用“IMU 先验增强跨视角鲁棒性”之类表述

### 10.6 训练更复杂，但收益很小
默认表述：
- 需要评估这种蒸馏是否具有实际价值
- 不能默认“只要涨一点就是有效改进”

### 10.7 某些 teacher target 有效，某些无效
默认表述：
- 说明不同 IMU 相关知识对 gait 识别的迁移价值不同
- 这类结果通常比“全都蒸馏”更有研究意义

## 15. 最终原则
分析“IMU 蒸馏到 gait”时，始终围绕下面三件事：

1. 当前结果是否真的支持 IMU 教师知识被迁移到了纯视觉 gait 学生；
2. 当前提升是否真的来自结构/时序先验，而不是普通训练收益；
3. 下一步最该补什么实验，才能把“看起来有效”变成“证据更充分”。

只要证据不足，就保守表达，不做过度解释。
